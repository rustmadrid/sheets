This library provides a Rust interface on the Wayland client library.

To use it, you'll need to have libwayland-client.so available on your system. However, to allow for easy optionnal support of wayland in applications, the library is not linked, but will be opened at first use of a wayland method.

You can check the presence of the library with the function is_wayland_lib_available(), and the methods creating the Display object will not panic if the library is absent, but return a failure (None or Err(_)).

The module core provides support for the core features of the wayland protocol. Some protocol extentions are available, each as their own module. Some of them require a system library which they will try to load at first use.

  • module egl: it provides a mean to build EGL surfaces in a wayland context. It requires the presence of libwayland-egl.so.
